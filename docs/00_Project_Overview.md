# 00 – Project Overview

## Project Name

**Family Heritage Platform**

## One-Line Description

A secure, multi-branch family history and genealogy platform that allows extended families to collaboratively build, manage, and explore their heritage across generations.

## Purpose

To provide a private, family-owned alternative to public genealogy services — one where the family controls the data, manages access, preserves privacy for living members, and maintains a permanent, multi-generational record of their heritage.

## Core Capabilities

| Capability | Description |
|-----------|-------------|
| Family Tree | Interactive graph-based tree with zoom, pan, and branch navigation |
| Person Profiles | Full profiles with photos, biography, addresses, occupations, and events |
| Relationships | Parent, child, spouse, adoption, step-family, guardian, and more |
| Branch Ownership | Each major family branch has its own admin |
| Media Library | Photos, videos, documents, certificates, letters, audio |
| Address History | Current and historical addresses with map visualization |
| Stories & Memories | Personal stories, oral history, and historical notes |
| Events | Births, marriages, deaths, migrations, and life milestones |
| Reports | PDF family books, ancestor reports, printable trees |
| AI Features | OCR, duplicate detection, relationship suggestions, biography generation |
| Security | Role-based access, privacy levels, audit trail, approval workflow |
| Mobile App | iOS and Android access for family members |

## Key Design Principles

1. **Privacy First** — Living members' data is protected by default
2. **Approval Workflow** — No change is published without review
3. **Nothing is Deleted** — Full audit trail and soft deletes only
4. **Graph Model** — Relationships are a graph, not a simple tree
5. **Branch Autonomy** — Branch Admins manage their own sub-family
6. **AI-Assisted** — AI helps but humans approve
7. **Mobile Ready** — Designed for both web and mobile from day one

## Target Users

| Role | Description |
|------|-------------|
| Super Admin | Owns the entire database, manages all branches |
| Branch Admin | Manages one family branch, approves edits |
| Member | Can view, suggest edits, upload media |
| Viewer | Read-only access (for guests or distant relatives) |
